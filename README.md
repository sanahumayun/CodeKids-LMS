# Learning Management System (LMS)

A role-based Learning Management System built for a startup use case, supporting Admins, Tutors, and Students with clearly separated permissions, structured course workflows, assignments, evaluations, and notifications.

---

## Core Features Overview

The LMS is built around three user roles, each with dedicated functionality and access control. The system supports the full course lifecycle â€” from course creation to completion and evaluation.

---

## User Roles

- Admin  
- Tutor  
- Student  

Each role has access to a personalized dashboard and role-specific actions.

---

## Admin Features

Admins have full system-level control and oversight.

- User Management
  - Create, read, update, and delete user profiles
  - Search for users across the platform

- Course Management
  - Create, read, update, and delete courses
  - View detailed course pages

- Course Oversight
  - View all assignments uploaded by tutors
  - View classwork submissions uploaded by students
  - View grades assigned by tutors for each assignment

- Evaluation and Approval
  - Review evaluation reports submitted by tutors
  - Approve evaluation reports for enrolled students
  - Mark courses as completed after evaluation approval

- Course Reviews
  - View student-submitted course ratings and reviews after course completion

---

## Tutor Features

Tutors manage academic content and student evaluations for assigned courses.

- Course Access
  - View courses they are assigned to

- Assignments
  - Upload assignments for a course
  - View and download student submissions
  - Grade assignments submitted by students

- Course Material
  - Upload learning materials for students

- Evaluations
  - Submit evaluation reports for enrolled students
  - Set a course to completed after evaluations are finalized, subject to admin approval

---

## Student Features

Students interact with course content and complete assigned work.

- Course Content
  - View course materials uploaded by tutors

- Classwork and Assignments
  - Upload classwork
  - Submit assignments for evaluation

- Course Feedback
  - Rate and review a course once it is completed and approved by the admin

---

## Dashboards

Each user role has access to a role-specific dashboard displaying relevant system statistics, such as:

- Number of enrolled students
- Number of courses
- Assignment submission activity
- Course status updates

---

## Notifications System

The platform includes an in-app notification system to keep users informed of important events, including:

- Assignment submissions by students
- Course completion updates
- Evaluation status changes
- Other role-specific system events

---

## Role-Based Access Control

- Strict permission handling ensures that users can only access features relevant to their role
- Admin, Tutor, and Student actions are fully isolated and secured
